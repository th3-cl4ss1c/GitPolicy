# Единая политика для агентов и участников

## Авто-коммит и авто-пуш

После каждого значимого изменения агент обязан:
1) прогнать проверки проекта;
2) сделать отдельный коммит по политике сообщений;
3) сделать push в `origin/main`;
4) в ответе указать hash коммита и результат push.

### Что считается значимым изменением

- изменены файлы в `src/`, `schemas/`, `tools/`, `.githooks/`, `README.md`, `AGENTS.md`;
- или изменено >= 20 строк кода/конфигурации/документации.

### Исключение

Если пользователь явно указал: "без коммита/пуша".

## Политика сообщений коммитов

- Сообщения коммитов на русском.
- Формат:

```text
<что было не так>
<что изменено>
```

- Рекомендуется добавлять:
- `Проверка:`
- `Риски:`

## Проверки и push

- Перед push выполняется легкая проверка: `bash -n run.sh`.
- Если нужно пропустить локальный `pre-push` хук: `SKIP_PRE_PUSH=1 git push ...`.
